INFORMACIÓN SOBRE MEMORY TRACE:
Contiene la traza de memoria en un formato legible. Cada línea se corresponde con un acceso a memoria, definido
por una dirección, el número de bytes accedidos y el tipo de acceso: fetch para acceso a código, read para 
lectura de datos y write para escritura de datos. Una restricción a tener en cuenta es que el simulador supone 
que cada dirección de memoria hace siempre referencia a 1 byte.

Cada conjunto de caché aparece numerado en la columna set
Los campos v, d, a y label son metadatos de la caché. Los datos que almacena realmente, asociados a las 
direcciones de memoria cacheadas son los que aparecen con los valores XXh
Cada linea de caché almacena 8 bytes

Los fallos de caché se muestran de varias formas. En primer lugar, con el borde rojo en la línea de la traza simulada. 
También se puede ver en los bytes de datos de la línea de caché afectada, pues aparecen en fondo rojo indicando 
que ha variado su contenido, tal como ocurre en un fallo de caché. Finalmente, en el panel de estadísticas se 
muestra en rojo el número de fallos de caché si en el último acceso se ha producido un fallo.

Ejemplo de descomposiciónde un acceso a memoria en campos de etiqueta, conjunto y desplazamiento:
Dirección: 0804h -> 000 1000 00//00 0//100b
Etiqueta (los 9 bits más significativos): 0 0010 0000b -> 020h
Conjunto (3 bits): 000b  -> 0h
Desplazamiento (los 3 bits menos significativos): 100b -> 4h
(el número de bits lo sabemos por el adress bits dado)

¿Cuál es la tasa de aciertos de la traza con la configuración de caché dada?
- Coger el valor del total hit ratio
---------------------------------------------------------------------------------------------------------------------
Valor B de la fórmula para calcular el tiempo medio de acceso a memoria se considera como el número de accesos 
a memoria necesarios para transferir la línea caché. Este número de accesos depende por tanto: del ancho de línea 
y del bus de datos utilizado para transferir la información

El tiempo necesario para satisfacer un fallo de caché sería el producto del tiempo de acceso a memoria principal 
por el número de accesos
 
se supone un bus de datos de 64 bits, un tiempo de acceso a lamemoria principal de 8 ciclos, y la memoria caché 
se ha configurado con un ancho de línea de 8 bytes. Conestos datos, el tiempo necesario para satisfacer un fallo 
de caché (Cache miss time en el simulador) sería:
Main memory access time x Line size / Bus size (en bytes). Esto es igual a 8 ciclos x 8 bytes / 8 bytes = 8
ciclos.
El tiempo medio de acceso a la jerarquía de memoria con los niveles de, memoria caché y memoria principal viene 
dada por la expresión:
T = Ac * tc + (1-Ac)*(tp*B)
Donde, Ac es la tasa de aciertos en caché; tc es el tiempo de acceso a caché; tp es el tiempo de acceso a memoria 
principal; y B es el tamaño de línea expresado en bytes.
La respuesta correcta es: 1,2982
